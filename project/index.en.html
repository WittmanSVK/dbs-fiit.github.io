Project

Assignment

Create a database application in an environment of your choice, which addresses at least 6 use case scenarios in their full complexity according to dmain of your choice. Exact scope and detailed description of selected scenarios is to be consulted and confirmed by your TA during lab sessions. You will create the application in two iterations. In the first one, you will be using relational database and provide implementation of following generic scenarios:

  • Creation of a new record,
  • Update of an existing record,
  • Deletion of a record,
  • Display of an overview of several records (along with a selected basic statistics),
  • Display of a record detail,
  • Filtering of records according user-provided criteria.

Application can have a console or graphical user interface. Essential is that chosen scenarios are realized "for real", i.e., application must provide the functionality in a way, which would be expected by a customer in given domain.

Scenarions, which are altering the data, must be realized using transactions and at least one of them must work with multiple tables (typically record creation and foreign key binding).

In secord iteration, you will add at least one additional scenario into the application, which will be based on using non=relational database Redis or Elasticsearch (or any other non-relational database, if agreed upon with your TA). Detailed description of the scenario is to be consulted and confirmed by your TA, depending on chosen database and your application domain (e.g., statistics on interaction with individual records using Redis or advanced search within records using Elasticsearch).

You need to have first iteration submitted and accepted to be eligible for submission of second iteration.

You need to have both iterations submitted and accepted to be eligible for final exam.

Implementation environment

Officially supported implementation stack is JAVA and PostgreSQL database server. Use of any other implementation environment is to be confirmed by your TA and it is expected that the student has previous knowledge of this environment (it is not appropriate to learn a new programming language within DBS course).

If a student decides to develop a web-based application, it is mandatory to use an appropriate web framework (MVC).

Evaluation & Marking

By developing the application you prove acquisition of practical skills of working with databases.

We evaluate:

  • Design of data model
  • Implementation of the model (physical data model)
    • it must be complete
  • Functionality of the application (i.e., agreed-upon scenarios)
  • Efficient use of relational database / SQL language
    • You must use JOIN in the application (everywhere, where it is appropriate, it must be appropriate at least once)
    • You must use an aggregation function in the application
    • You must use GROUP BY in the application
  • Software design and user interface ergonomy

Marking:

  • First iteration - total 18 pts.
    • Functionality - 9/18
    • Documentation - 3/18
    • Quality (model, code) - 6/18
  • Second iteration - total 12pts.
    • Functionality - 6/12
    • Documentation - 2/12
